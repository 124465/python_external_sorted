# 1. external_sorted

Этот файл используется для внешней сортировки.

## Команда

```bash
py external_sort.py random_numbers.txt 1000000 --workers 8
```

## Объяснение параметров

- `random_numbers.txt` — путь к входному файлу (текстовый файл, где каждое число находится на отдельной строке).
- `1000000` — максимальное количество целых чисел, одновременно загружаемых в память (контроль использования памяти).
- `--workers 8` — использование 8 процессов для параллельной сортировки блоков (задействуются многоядерные CPU).
- Вывод: по умолчанию в той же директории создаётся файл `random_numbers_sorted.txt` (сортировка по возрастанию).

---

## Анализ сложности

Общая временная сложность внешней сортировки (`external sort`) состоит из двух этапов.

### Обозначения

- `n` — общее количество чисел
- `m` — максимальное количество чисел, помещающихся в память за один раз
- `r ≈ ⌈n/m⌉` — количество отсортированных блоков
- `k` — количество файлов, объединяемых одновременно при многопутевом слиянии (`k ≤ max_open_files`)

---

### 1. Генерация отсортированных блоков (Run Generation)

Файл делится на `r` частей, каждая размером не более `m`, после чего каждая часть сортируется в памяти.

- Сортировка одного блока: `O(m log m)`
- Всего блоков: `r`

Итоговая сложность:

```text
O(r · m log m) = O(n log m)
```

---

### 2. Многопутевое слияние (Merge)

Все `r` отсортированных блоков объединяются в один файл.

#### Если возможно выполнить `r`-путевое слияние за один проход (`k ≥ r`)

Сложность:

```text
O(n log r)
```

#### Если `k` ограничено

Требуется несколько проходов:

```text
Количество проходов ≈ ⌈log_k r⌉
```

Общая сложность:

```text
O(n · log k · ⌈log_k r⌉) = O(n log r)
```

---

### Общая временная сложность

```text
O(n log m + n log r) = O(n log n)
```

Так как:

```text
m · r ≈ n
```

следовательно:

```text
log m + log r = log n
```

Таким образом, алгоритм удовлетворяет требованию:

```text
Временная сложность не превышает O(n log n)
```

# 2. print_head

Выводит первые `N` чисел из отсортированного файла (по умолчанию — 20).

## Команда

```bash
py print_head.py random_numbers_sorted.txt --n 20
```

## Объяснение параметров

- `py` — Python launcher для Windows
- `print_head.py` — скрипт для чтения файла и вывода первых `N` чисел
- `random_numbers_sorted.txt` — отсортированный файл
- `--n 20` — вывести первые 20 чисел (если параметр не указан, по умолчанию используется 20)

---

## Вывод

По умолчанию результат выводится в консоль.

Также можно сохранить результат в файл:

```bash
py print_head.py random_numbers_sorted.txt --n 20 --output head20.txt
```

---

## Вывод последних N чисел

Для вывода последних `N` чисел используйте параметр `--tail`:

```bash
py print_head.py random_numbers_sorted.txt --n 20 --tail
```

# 3. print_tail

Выводит последние `N` чисел из отсортированного файла (по умолчанию — 20).

## Команда

```bash
py print_tail.py random_numbers_sorted.txt --n 20
```

## Объяснение параметров

- `print_tail.py` — специальный скрипт для вывода последних `N` чисел
- `--n` — количество выводимых чисел
- `--output` — сохранить результат в файл вместо вывода в консоль

---

## Примечание

Внутри `print_tail.py` фактически вызывает `print_head.py` с автоматически добавленным параметром `--tail`.
